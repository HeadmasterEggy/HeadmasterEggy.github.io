COMP5313 Tutorial 05 — Hub and Authority 详细题解

HITS 算法回顾

两个核心向量：

  • Authority 分值 \(a(v)\)：有多少高质量 Hub 指向 \(v\)
  • Hub 分值 \(h(v)\)：\(v\) 指向了多少高质量 Authority

两步迭代（\(k=2\) 表示做两轮）：

  1. Authority update：\(a(v) \leftarrow \sum_{(u \to v)} h(u)\)
  2. Hub update：\(h(v) \leftarrow \sum_{(v \to w)} a(w)\)

注意：初始时，只有叶子节点（无出链的节点，如 A、B）作为 Authority 的入度来源。HITS 通常对所有节点初始化 \(h=1\)，然后归一化。

图 1 结构（关键）

有向边（尾 → 头表示链接方向）：

起点 终点
C A
D A
E A, B
F B
A C, D, E
B F

验证 Round 1： - \(auth(A) = h(C) + h(D) + h(E) = 1+1+1 = 3\) ✓ - \(auth(B) = h(E) + h(F) = 1+1 = 2\) ✓ - \(hub(C) = a(A) = 3\)（C → A）、\(hub(D) = a(A) = 3\)、\(hub(E) = a(A)+a(B) = 5\)（E → A 且 E → B）、\(hub(F) = a(B) = 2\) ✓

Exercise 1：两轮 HITS 计算

初始化

\[h(C) = h(D) = h(E) = h(F) = 1, \quad h(A) = h(B) = 0\] \[a(*) = 0\]

Round 1

Authority update：

\[a(A) = h(C) + h(D) + h(E) = 1+1+1 = 3\] \[a(B) = h(E) + h(F) = 1+1 = 2\]

Hub update：

\[h(C) = a(A) = 3 \quad (\text{C 的出链指向 A})\] \[h(D) = a(A) = 3\] \[h(E) = a(A) + a(B) = 3+2 = 5 \quad (\text{E 的出链指向 A 和 B})\] \[h(F) = a(B) = 2 \quad (\text{F 的出链指向 B})\]

Round 2

Authority update：

\[a(A) = h(C) + h(D) + h(E) = 3+3+5 = 11\] \[a(B) = h(E) + h(F) = 5+2 = 7\]

Hub update：

\[h(C) = a(A) = 11\] \[h(D) = a(A) = 11\] \[h(E) = a(A) + a(B) = 11+7 = 18\] \[h(F) = a(B) = 7\]

归一化（Normalization）

Authority 归一化（除以 \(11+7=18\)）：

\[\hat{a}(A) = \frac{11}{18}, \quad \hat{a}(B) = \frac{7}{18}\]

Hub 归一化（除以 \(11+11+18+7=47\)）：

\[\hat{h}(C) = \hat{h}(D) = \frac{11}{47}, \quad \hat{h}(E) = \frac{18}{47}, \quad \hat{h}(F) = \frac{7}{47}\]

Exercise 2：添加新页面 X 和 Y

Case 1：Y 只指向 X

新图边：在原图基础上增加：Y → X（X 无出链）

Round 1 和 Round 2 中，Y 的 hub 分值和 X 的 auth 分值始终等于 1（因为 X 无出链，Y 只指向 X）。

归一化后（Round 2 总 auth = 19，总 hub = 48）：

\[\hat{h}(Y) = \frac{1}{48}, \quad \hat{a}(X) = \frac{1}{19}\]

Case 2：Y 指向 A、B 和 X

新图：

Round 1：

\[a(A) = h(C)+h(D)+h(E)+h(Y) = 1+1+1+1 = 4\] \[a(B) = h(E)+h(F)+h(Y) = 1+1+1 = 3\] \[a(X) = h(Y) = 1\]

\[h(C)=h(D)=a(A)=4, \quad h(E)=a(A)+a(B)=7, \quad h(F)=a(B)=3, \quad h(Y)=a(A)+a(B)+a(X)=8\]

Round 2：

\[a(A) = h(C)+h(D)+h(E)+h(Y) = 4+4+7+8 = 23\] \[a(B) = h(E)+h(F)+h(Y) = 7+3+8 = 18\] \[a(X) = h(Y) = 8\]

归一化（总 auth = \(23+18+8=49\)）：

\[\hat{a}(A) = \frac{23}{49}, \quad \hat{a}(B) = \frac{18}{49}, \quad \hat{a}(X) = \frac{8}{49}\]

Exercise 3：X 排名第二的策略

策略（图 3）

核心思路：

  1. 让 Y 和 Z 都指向 A（而非 B）—— 这样 Y 和 Z 能获得最大 hub 分值（因为 A 有三个 hub 出链 C、D、E，auth(A) 会很大）
  2. 让 Y 和 Z 同时指向 X，把 hub 分值转化为 X 的 authority 分值

为什么 X 不能排名第一？

HITS 两轮后，\(auth(X) \leq hub(Y) + hub(Z)\)。

  • 若 Y、Z 都连向 A：hub(Y) 和 hub(Z) 会流向 auth(A)，X 最多排第二
  • 若 Y、Z 都连向 B：同理，X 也超不过 B
  • 若 Y、Z 都不连 A 或 B：hub(Y) 和 hub(Z) 很小，auth(X) 也无法达到最大

因此，X 能达到的最高名次是第二，这是 Exercise 3 策略能实现的上界。
